Strategy Pattern

 Posted on January 1, 0001  |  4 minutes  |  640 words  |  Kunlei Lian

Strategy Pattern, also known as policy pattern, is one of the behavioral design patterns that promote software flexibility and reusability. It features in allowing classes to select algorithms at runtime by encapsulating algorithms and making them interchangable.

This pattern is better explained through an example. Imagine we are tasked to model a group of kids within a preschool. Everyone can sing and jump, but naturally the way they sing and jump vary kid by kid. For simplicity, let’s say a child can either sing loudly or quietly, and either jump high or low. The strategy pattern defines separate interfaces for these behaviors:

1public interface Sing {
2    void sing();
3}
1public interface Jump {
2    void jump();
3}

Based on the possible ways of singing and jumping, various implementations can be possible:

1public class SingAloud implements Sing {
2    @Override
3    public void sing() {
4        System.out.println("I am singing aloud!");
5    }
6}
1public class SingQuiet implements Sing {
2    @Override
3    public void sing() {
4        System.out.println("I am singing quietly...");
5    }
6}

The same is true for jumping:

1public class JumpHigh implements Jump {
2    @Override
3    public void jump() {
4        System.out.println("I can jump very high!");
5    }
6}
1public class JumpLow implements Jump {
2    @Override
3    public void jump() {
4        System.out.println("I can only jump low...");
5    }
6}

Having defined the possible behaviors, we can go ahead and create a base class for children within the group:

 1public abstract class Child {
 2    protected String name;
 3    protected Sing sing;
 4    protected Jump jump;
 5
 6    public void setSing(Sing sing) {
 7        this.sing = sing;
 8    }
 9
10    public void setJump(Jump jump) {
11        this.jump = jump;
12    }
13
14    public void sing() {
15        sing.sing();
16    }
17    public void jump() {
18        jump.jump();
19    }
20}

Note that this base class is decoupled fron concrete behaviors, it only stores a reference to the singing behavior and jumping behavior. In other words, it lets its subclasses to decide which algorithm to use at runtime. Let’s look at some examples:

1public class XiaoMing extends Child {
2    public XiaoMing() {
3        name = "Xiao Ming";
4        setSing(new SingAloud());
5        setJump(new JumpHigh());
6    }
7}
1public class GouDan extends Child {
2    public GouDan() {
3        name = "Gou Dan";
4        setSing(new SingQuiet());
5        setJump(new JumpLow());
6    }
7}
1public class ErWa extends Child {
2    public ErWa() {
3        name = "Er Wa";
4        setSing(new SingQuiet());
5        setJump(new JumpHigh());
6    }
7}

Now let’s watch children playing in the classroom. Note that each child has his/her own default singing/jumping behavior at initialization. The interesting part is that they can change their minds at runtime!

 1public class ChildrenAtPlay {
 2
 3    public static void main(String[] args) {
 4        // let's first define singing and jumping algorithms
 5        SingAloud singAloud = new SingAloud();
 6        SingQuiet singQuiet = new SingQuiet();
 7        JumpHigh jumpHigh = new JumpHigh();
 8        JumpLow jumpLow = new JumpLow();
 9
10        // here are some kids in the group
11        XiaoMing xiaoMing = new XiaoMing();
12        GouDan gouDan = new GouDan();
13        ErWa erWa = new ErWa();
14
15        // children at play
16        xiaoMing.sing();
17        xiaoMing.jump();
18        gouDan.sing();
19        gouDan.jump();
20        erWa.sing();
21        erWa.jump();
22
23        // children change their minds all the time
24        xiaoMing.setSing(singQuiet);
25        xiaoMing.setJump(jumpLow);
26        gouDan.setSing(singAloud);
27        gouDan.setJump(jumpHigh);
28
29        // children at play again
30        System.out.println();
31        xiaoMing.sing();
32        xiaoMing.jump();
33        gouDan.sing();
34        gouDan.jump();
35        erWa.sing();
36        erWa.jump();
37    }
38}
