Observer Pattern

   |  3   |  557   |  Kunlei Lian

Observer Pattern is a behavioral pattern suitable for decoupling objects with one-to-many relations. There are two components in this pattern:

  • Subject: this is the object whose state is subject to changes and will affect many following objects.
  • Observer: this is the object which will updates its status based on the status of the subject.

An example is the information updating system used in my daughter’s school. Parents get udpates from the class teacher constantly.

1public interface Subject {
2    void registerObserver(Observer o);
3    void removeObserver(Observer o);
4    void notifyObservers();
5}
1public interface Observer {
2    void update(String status);
3}
 1/**
 2 * Subject implementation
 3 * @author Kunlei Lian
 4 */
 5public class Teacher implements Subject {
 6    private String classStatus;
 7    private List<Observer> observers;
 8
 9    public Teacher() {
10        observers = new ArrayList<>();
11    }
12
13    public void updateClassStatus(String time) {
14        switch (time) {
15            case "morning":
16                classStatus = "Kids are playing in classroom!";
17                break;
18            case "noon":
19                classStatus = "Kids are having nap time!";
20                break;
21            case "afternoon":
22                classStatus = "Kids are having fun on playground!";
23                break;
24            default:
25                classStatus = "Kids are idle....";
26                break;
27        }
28        notifyObservers();
29    }
30
31    @Override
32    public void registerObserver(Observer o) {
33        observers.add(o);
34    }
35
36    @Override
37    public void removeObserver(Observer o) {
38        observers.remove(o);
39    }
40
41    @Override
42    public void notifyObservers() {
43        for (Observer o : observers) {
44            o.update(classStatus);
45        }
46    }
47}
 1/**
 2 * Observer implementation
 3 * @author Kunlei Lian
 4 */
 5public class Parent implements Observer {
 6    private String name;
 7    private Subject teacher;
 8
 9    public Parent(String name, Subject teacher) {
10        this.name = name;
11        this.teacher = teacher;
12        this.teacher.registerObserver(this);
13    }
14
15    @Override
16    public void update(String status) {
17        System.out.println(name + " got update from Teacher: " + status);
18    }
19
20    public void exitProgram() {
21        teacher.removeObserver(this);
22    }
23}

Let’s run a demo:

 1/**
 2 * Preschool information updating system demo
 3 * @author Kunlei Lian
 4 */
 5public class ObserverDemo {
 6
 7    public static void main(String[] args) {
 8        // a subject with two observers
 9        Teacher teacher = new Teacher();
10        Parent parenta = new Parent("Parent AA", teacher);
11        Parent parentb = new Parent("Parent BB", teacher);
12
13        // subject broadcasts events to observers
14        System.out.println();
15        teacher.updateClassStatus("morning");
16        teacher.updateClassStatus("noon");
17        teacher.updateClassStatus("afternoon");
18
19        // new observer
20        System.out.println();
21        Parent parentc = new Parent("Parent CC", teacher);
22        teacher.updateClassStatus("morning");
23
24        // existing observer stop observing
25        System.out.println();
26        parentb.exitProgram();
27        teacher.updateClassStatus("noon");
28    }
29}
